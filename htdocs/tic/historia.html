As implementações de coleta nas versões pré-JDK 1.2 da plataforma Java incluíam poucas classes de estrutura de dados, mas não continham uma estrutura de coletas.

Os métodos padrão para agrupar objetos Java eram através da matriz, do Vetor e das classes Hashtable, que infelizmente não eram fáceis de estender, e não implementavam uma interface padrão de membro.

Para atender à necessidade de estruturas de dados de coleta reutilizáveis, várias estruturas independentes foram desenvolvidas, sendo o pacote de Coleções de Doug Lea, e a ObjectSpace Generic Collection Library (JGL), cujo principal objetivo era a consistência com a Biblioteca de Modelos Padrão C++(STL).

A estrutura de coleções foi projetada e desenvolvida principalmente por Joshua Bloch, e foi introduzida no JDK 1.2. Ele reapriu muitas ideias e aulas do pacote Coleções de Doug Lea, que foi preterido como resultado. A Sun Microsystems optou por não usar as ideias da JGL, porque queriam uma estrutura compacta, e a consistência com o C++ não era um de seus objetivos.

Doug Lea mais tarde desenvolveu um pacote de concorrência, compreendendo novas classes relacionadas à coleção. Uma versão atualizada desses utilitários de concorrência foi incluída no JDK 5.0 a partir de JSR 166.
